# 用户资料卡撰写指引

在任何志愿推荐前，先把用户信息整理成本地 Markdown 资料卡。资料卡是后续查询、推荐、排序、复核和最终填表的唯一用户画像基准。

## 存放位置

将资料卡写入项目根目录的任务产物目录，文件名固定为 `profile-card.md`：

```text
output/{考生代号}-{科类}-{位次}/profile-card.md
```

命令默认从项目根目录执行，输出路径写作 `output/{考生代号}-{科类}-{位次}/profile-card.md`。

如果用户未提供姓名，使用 `anonymous`。如果用户要求隐私保护，只保存匿名代号，不写身份证号、手机号、准考证号等敏感信息。

## 必填信息

资料卡至少包含：

- 姓名或匿名代号
- 年份：2026
- 科类：物理类、历史类、艺术类、体育类，或 3+证书/学考等
- 高考分数
- 全省排位/位次
- 目标批次：本科、专科、提前批、艺体、春考等
- 选科组合或专业统考类别
- 是否接受专业调剂
- 是否有体检、单科、语种、性别、色盲色弱、身高等限制

## 重要偏好

尽量收集并结构化：

- 院校偏好：倾向院校、院校层次、公办/民办、中外合作接受度
- 地域偏好：广东省内、省外、城市名单、不能接受地区
- 专业偏好：喜欢专业、排斥专业、专业门类、是否接受大类招生
- 学习偏好：如不想学数学、害怕化学实验、不接受医学长学制等
- 家庭经济：可接受年学费区间，是否排斥高收费、中外合作、国际班
- 职业规划：考公、考研、就业、出国、进体制、进大厂、家族行业等
- 风险偏好：冲刺意愿、是否优先保本科、是否宁可学校降档保专业

## 使用规则

- 在每轮推荐前重新读取资料卡。
- 正式推荐前优先使用 `python <skill_dir>/scripts/create_profile_card.py --input output/{考生代号}-{科类}-{位次}/candidate.json --output output/{考生代号}-{科类}-{位次}/profile-card.md --strict`；如果严格校验缺少科类、位次、目标批次、选科/专业统考类别、是否服从调剂、体检或单科/语种限制等核心字段，必须先追问用户。
- 当用户补充或改变偏好时，先更新资料卡，再继续筛选。
- 不要把用户明确排斥的专业、地区、收费类型放入稳、保、垫、兜底层。
- 对用户排斥但可能有策略价值的选项，只能放入“备选解释”，不能直接写入正式推荐清单。
