# 推荐工作流与输出清单

本文件规定如何从用户资料卡、政策规则和数据脚本生成“冲、稳、保、垫、兜底”院校专业推荐清单。

## 总流程

1. 读取用户资料卡。
2. 读取 `references/2026-policy-rules.md`。
3. 读取 `references/data-sources.md`。
4. 使用 `scripts/equivalent_score.py` 获取用户分数/位次在 2022-2025 的等位参考。
5. 使用 `scripts/query_admissions.py` 查询院校专业组、专业录取分、招生计划、章程链接。
6. 将候选项按用户偏好、风险层级、政策限制、经济限制过滤。
7. 生成结构化推荐清单。
8. 等用户确认后，再做最终交叉复核。
9. 用 `scripts/render_volunteer_draft.py` 填充志愿表草稿，配合 `assets/2026年广东省普通高校招生考生志愿表（官方）.pdf` 核对字段，并基于 `assets/2026广东高考志愿表填报模板.xlsx` 输出可编辑 Excel 文件。

## 信息不足时

如果缺少位次、科类、目标批次、是否艺体等核心信息，先向用户追问。不要用模糊信息直接生成正式推荐。

如果缺少兴趣、地域、经济、职业规划等软信息，可以先做“数据候选池”，但正式推荐前必须提醒用户补充偏好。

## 候选池构建

普通类建议先使用院校专业组层数据：

```bash
python scripts/query_admissions.py --source school-scores --year 2025 --subject 物理类 --batch 本科批 --rank-center 50000 --rank-width 20000 --limit 80
```

再用专业分和招生计划交叉：

```bash
python scripts/query_admissions.py --source major-scores --year 2025 --subject 物理类 --school-exact 中山大学 --limit 50
python scripts/query_admissions.py --source plans --year 2025 --subject 物理类 --school-exact 中山大学 --limit 50
```

如果用户给专业偏好，先按专业关键词查专业层数据，再回到专业组层做风险判断：

```bash
python scripts/query_admissions.py --source major-scores --year 2025 --subject 物理类 --major-contains 计算机 --rank-center 50000 --rank-width 30000
```

如果查询具体院校，优先使用 `--school-code` 或 `--school-exact`。只有做模糊探索时才使用 `--school-contains`。

如果 `major-scores` 对目标院校返回 `No rows matched`，不要假设该校没有相关专业。改用 `plans` 查询专业代码、专业组、计划数和备注，并在推荐清单中标记“专业分缺失，已用招生计划补查”。

招生章程查询：

```bash
python scripts/query_admissions.py --source charters --school-exact 清华大学
```

艺术/体育类先用对应专区数据，再按 2026 合成分规则和最新专业目录复核。

## 风险层级

将候选项分为：

- 冲：历史最低位次明显优于用户位次，或等位分高于用户 3-20 分，录取不确定但有兴趣价值。
- 稳：历史最低位次接近用户位次，近年波动可控，专业组内可接受专业较多。
- 保：历史最低位次低于用户位次，计划稳定，退档风险可控。
- 垫：明显低于用户位次，用于降低滑档风险。
- 兜底：足够低且用户可接受；优先公办、专业可接受、调剂风险低。

不要机械套固定分差。必须结合：

- 2025 位次
- 2024/2023/2022 趋势
- 招生计划增减
- 专业组内专业构成
- 用户是否服从调剂
- 高收费/体检/单科/语种限制

## 推荐清单字段

正式清单必须包含：

- 层级：冲/稳/保/垫/兜底
- 院校名称
- 院校代码
- 专业组代码
- 批次
- 科类
- 推荐专业或专业方向
- 可接受专业组内其他专业
- 2025 最低分/最低位次
- 2024/2023/2022 参考位次或等位分
- 2026 招生计划核验状态
- 专业分核验状态：已查到专业分 / 专业分缺失，已用计划表补查
- 学费/收费风险
- 地域
- 办学性质
- 985/211/双一流或其他标签
- 推荐理由
- 风险提示
- 章程核查链接或待核查说明
- 是否建议服从调剂

## 反模式

不要这样做：

- 只按分数推荐，不看位次。
- 只按院校名称推荐，不看专业组。
- 只看目标专业，不看同组调剂专业。
- 把用户明确排斥的地区、专业、收费类型放入保底。
- 使用 2022-2025 历史数据冒充 2026 最终招生计划。
- 忽略体检、单科、语种、性别、色盲色弱等章程条件。
- 未经用户确认就直接生成最终志愿表。

## 用户确认前输出

先输出“推荐清单供审阅”，不要直接填最终志愿表。请用户确认：

- 是否接受每个层级的风险比例。
- 是否删除任何院校/专业。
- 是否加入新的院校、地区或专业偏好。
- 是否接受高收费、中外合作、联合培养等项目。
- 是否接受服从调剂。

## 最终复核

用户确认后，逐项复核：

- 用户资料卡是否更新到最新偏好。
- 目标批次是否正确。
- 志愿是否在正确时段填报。
- 专业组代码、专业代码、院校代码是否来自最新招生专业目录。
- 是否符合选科、合格考、体检、语种、单科和特殊资格要求。
- 同一批次内艺体/普通类是否冲突。
- 高水平运动队/综合评价是否二选一。
- 春季高考已录取考生是否只报允许类别。
- 志愿顺序是否存在倒挂。
